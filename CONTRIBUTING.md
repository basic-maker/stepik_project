# Contributing Guide

## Как вносить изменения в документацию

1. Создайте отдельную ветку от main для своих изменений.
2. Вносите правки в файлы документации, используя Markdown.
3. Делайте логичные и небольшие коммиты с понятными сообщениями.
4. Перед отправкой убедитесь, что документация корректно отображается.

## Процесс ревью

1. Создайте Pull Request в ветку main.
2. Опишите, какие изменения были внесены и зачем.
3. Дождитесь ревью и комментариев.
4. После одобрения выполните слияние изменений.
